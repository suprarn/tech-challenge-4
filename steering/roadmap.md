# ROADMAP: TECH CHALLENGE 4 - DATA ANALYTICS

## FASE 1: Consolidação da EDA 
* Limpeza e tratamento da base de dados obesity.csv.
* Análise exploratória focada em fatores genéticos, ambientais e comportamentais.
* Visualização de dados utilizando Seaborn e Matplotlib para identificar padrões de saúde.

## FASE 2: Engenharia de Atributos e Pré-processamento 
* Implementação de pipeline demonstrando feature engineering conforme exigência do projeto.
* Criação da variável BMI (IMC) para potencializar a predição do nível de obesidade.
* Tratamento de variáveis categóricas como Gênero, Histórico Familiar e Hábitos Alimentares.

## FASE 3: Modelagem e Treinamento
* Desenvolvimento do modelo de Machine Learning para auxílio médico.
* Garantia de assertividade do modelo acima de 75%, evitando toda e qualquer forma de Data Leak.
* Validação do pipeline de treinamento completo através de métricas de classificação.

## FASE 4: Sistema Preditivo e Dashboard (Streamlit)
* Realização do deploy do modelo em aplicação utilizando o framework Streamlit.
* Construção de painel analítico com insights para a equipe médica utilizando Seaborn e Matplotlib.
* Aplicação de visão de negócio tanto no sistema quanto no dashboard.

## FASE 5: Entrega e Apresentação
* Criação de um script para um vídeo de apresentação estratégico, cobrindo a metodologia e o sistema. A visão do sistema preditivo e o dashboard analítico devem ser apresentados em uma visão de negócio. (Descrito em um documento tipo markdown independente e exclusivo para isso)
* Organização do repositório GitHub com todo o código desenvolvido, o processo de construção do projeto é dividido em fases, sendo que cada fase deve ter o seu código independente, limpo, organizado e documentado.
* Geração de arquivo de entrega (.doc ou .txt) com os links da aplicação e do repositório.
